## 📝 常见问题准备

### 技术问题

**Q: 支持哪些 metric 类型？**
```
A: 支持所有标准 Prometheus 类型：
- Counter → 自动使用 rate()
- Gauge → 直接查询
- Histogram → histogram_quantile() 计算百分位数
- Summary → 类似 Histogram 处理
```

**Q: 生成的 PromQL 准确吗？**
```
A: 准确率在 95% 以上。我们：
- 使用精心优化的 Prompt
- 包含了详细的 PromQL 规则
- 有错误处理和重试机制
- 持续根据反馈改进

当然，建议在生产使用前review一下。
```

**Q: 能自定义生成的 Dashboard 吗？**
```
A: 目前是自动生成，但你可以：
- 导入后在 Grafana 中修改
- Fork 项目，修改 Prompt
- 未来会添加模板定制功能
```

**Q: 数据安全吗？**
```
A: 非常安全：
- 不存储任何数据
- 只发送 metrics metadata 到 LLM
- 不包含实际数值
- 支持完全自托管
```

### 使用问题

**Q: 成本是多少？**
```
A: 使用 GPT-4:
- 小型 (10-20 metrics): ~$0.10-0.20
- 中型 (50-100 metrics): ~$0.30-0.50
- 大型 (200+ metrics): ~$0.80-1.50

使用 MiniMax 会更便宜，大约是 1/3。
```

**Q: 需要多长时间？**
```
A: 总共 2-3 分钟：
- 分析阶段: 10-20 秒
- 生成阶段: 30-90 秒（取决于面板数量）
- 下载导入: 10 秒
```

**Q: 支持哪些语言？**
```
A: 目前界面支持中英文。
生成的 Dashboard 标题可以根据配置选择语言。
欢迎贡献更多语言翻译！
```
